# Счётчик просмотров

## Задача

Есть статья сайта. Каждый просмотр должен увеличивать счётчик:

```
article:10:views
```

Для реализации используется команда `INCR`, которая атомарно увеличивает значение ключа на 1.

---

## Увеличение счётчика

```
127.0.0.1:6379> INCR article:10:views
(integer) 1

127.0.0.1:6379> INCR article:10:views
(integer) 2

127.0.0.1:6379> INCR article:10:views
(integer) 3

127.0.0.1:6379> INCR article:10:views
(integer) 4

127.0.0.1:6379> INCR article:10:views
(integer) 5
```

## Получение текущего значения

```
127.0.0.1:6379> GET article:10:views
"5"
```

---

Ключ `article:10:views` используется как счётчик просмотров статьи.  
Команда `INCR` увеличивает значение атомарно, поэтому такой подход подходит для подсчёта просмотров, лайков, переходов и других событий.

# Рейтинг статей

## Задача

Необходимо создать leaderboard популярных статей.

Для этого используется структура данных Sorted Set.

Sorted Set хранит:

- элемент;
- числовой score;
- автоматически сортирует элементы по score.

В качестве ключа используется:

```
articles:leaderboard
```

---

## Добавление статей с количеством просмотров

```
127.0.0.1:6379> ZADD articles:leaderboard 120 article:1
(integer) 1

127.0.0.1:6379> ZADD articles:leaderboard 350 article:2
(integer) 1

127.0.0.1:6379> ZADD articles:leaderboard 90 article:3
(integer) 1

127.0.0.1:6379> ZADD articles:leaderboard 220 article:4
(integer) 1
```

| Статья    | Просмотры |
| --------- | --------: |
| article:1 |       120 |
| article:2 |       350 |
| article:3 |        90 |
| article:4 |       220 |

---

## Получить топ-3 статьи без количества просмотров

```
127.0.0.1:6379> ZREVRANGE articles:leaderboard 0 2
1) "article:2"
2) "article:4"
3) "article:1"
```

Топ-3 статей:

1. `article:2`
2. `article:4`
3. `article:1`

---

## Получить топ-3 статьи с количеством просмотров

```
127.0.0.1:6379> ZREVRANGE articles:leaderboard 0 2 WITHSCORES
1) "article:2"
2) "350"
3) "article:4"
4) "220"
5) "article:1"
6) "120"
```

Результат показывает статьи вместе со score, то есть количеством просмотров.

---

## Добавить большое количество просмотров статье

Добавим `500` просмотров статье `article:1`:

```
127.0.0.1:6379> ZINCRBY articles:leaderboard 500 article:1
"620"
```

Теперь у `article:1` стало `620` просмотров.

---

## Новый топ-3

```
127.0.0.1:6379> ZREVRANGE articles:leaderboard 0 2 WITHSCORES
1) "article:1"
2) "620"
3) "article:2"
4) "350"
5) "article:4"
6) "220"
```

После увеличения просмотров `article:1` поднялась на первое место.

# Ограничение действий пользователя

## Задача

Пользователь может поставить максимум 5 лайков за минуту.

Для этого используется ключ:

```
user:100:likes
```

Идея:

- каждый лайк увеличивает счётчик;
- ключ живёт 60 секунд;
- после истечения TTL ключ удаляется;
- если значение больше 5 — действие можно ограничить.

---

## Увеличение счётчика лайков

```
127.0.0.1:6379> INCR user:100:likes
(integer) 1

127.0.0.1:6379> INCR user:100:likes
(integer) 2

127.0.0.1:6379> INCR user:100:likes
(integer) 3

127.0.0.1:6379> INCR user:100:likes
(integer) 4

127.0.0.1:6379> INCR user:100:likes
(integer) 5
```

---

## Установка TTL 60 секунд

```
127.0.0.1:6379> EXPIRE user:100:likes 60
(integer) 1
```

---

## Проверка текущего значения

```
127.0.0.1:6379> GET user:100:likes
"5"
```

---

## Проверка оставшегося времени жизни ключа

```
127.0.0.1:6379> TTL user:100:likes
(integer) 54
```

Это означает, что ключ будет автоматически удалён примерно через 54 секунды.

---

## Попытка поставить ещё один лайк

```
127.0.0.1:6379> INCR user:100:likes
(integer) 6
```

Текущее значение стало больше допустимого лимита.

Проверка:

```
127.0.0.1:6379> GET user:100:likes
"6"

если user:100:likes > 5, то действие запрещается до истечения TTL
```

---

## Проверка TTL после дополнительного лайка

```
127.0.0.1:6379> TTL user:100:likes
(integer) 48
```

TTL продолжает уменьшаться. Ключ будет удалён автоматически по истечении 60 секунд с момента установки TTL.

---

## Проверка после истечения TTL

Через минуту:

```
127.0.0.1:6379> GET user:100:likes
(nil)
```

Ключ удалён, лимит для пользователя сброшен.
